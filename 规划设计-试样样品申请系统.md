## 一、场景概述

### 1.1 业务定义

**试样样品申请系统** 是一套面向 **跨行业客户** 的样品申请全流程数字化管理平台。

其核心价值在于：
- **多行业适配**：不同行业（汽车零部件、精密制造、电子元件等）可自定义申请表单字段
- **全流程可追溯**：从提交 → 审批 → 制作/发货 → 完成，每个状态变更都有时间、操作人记录
- **客户可查询**：客户通过申请编号或公司信息，自行追踪样品进度

## 二、适用行业

### 2.1 核心行业

| 行业 | 典型样品类型 | 特点 |
|---|---|---|
| **汽车零部件** | 总成试样、耐久测试样件 | 需 OE 号、零件号、材料标准、PV 测试 |
| **精密制造/机械加工** | 零部件试样、模具试样 | 需精度等级、材质要求、表面处理 |
| **电子/半导体** | PCB 样板、芯片测试样 | 交期敏感，需规格参数 |
| **化工/新材料** | 配方试样、改性材料 | 需附检测报告、安全数据表 |
| **医疗器械** | 生物试样、植入物试样 | 合规要求高，需追溯完整 |
| **消费品/日化** | 包装样品、配方样品 | 市场驱动，时效性强 |
| **食品/农产品** | 原料样品、成品样品 | 保质期敏感 |
| **纺织/服装** | 面料样、色卡样 | 季节性强 |
| **建材/五金** | 型材试样、涂层样 | 批量小，规格多 |

### 2.2 核心能力

后台可自由创建行业，每个行业配置专属字段（文本、下拉、数字、日期、开关等），前台自动同步展示对应表单。

## 三、行业痛点

| 痛点 | 描述 | 后果 |
|---|---|---|
| **P1. 流程黑盒** | 申请人提交后不知道进度 | 沟通成本高，影响交期 |
| **P2. 单据散落** | 纸质单、邮件、微信多方散落 | 历史数据不可查 |
| **P3. 多行业表单不统一** | 不同行业样品字段差异大 | 信息不全，重复沟通 |
| **P4. 审批无标准** | 缺少时效要求 | 申请被延误 |
| **P5. 重复申请** | 无样品台账 | 资源浪费 |
| **P6. 数据断层** | 申请、制作数据分散 | 无法追溯全生命周期 |

## 四、功能全景

### 4.1 前台功能

| 功能 | 说明 |
|---|---|
| **样品申请表单** | 动态表单，根据后台行业配置展示专属字段 |
| **产品清单** | 支持添加多个产品项目 |
| **申请人信息** | 公司名称、姓名、职务、联系方式、邮箱、地址 |
| **文件附件上传** | 支持图纸、技术文档、照片等 |
| **申请编号查看** | 提交成功后展示唯一编号 |
| **查询申请进度** | 按申请编号或公司+联系方式查询状态与进度追踪 |
| **草稿保存** | 未提交的表单可暂存草稿，下次恢复 |
| **提交时间** | 记录每次提交的具体时间 |

### 4.2 后台功能

| 功能 | 说明 |
|---|---|
| **样品申请台账** | 查看所有申请记录、搜索、筛选、导出 CSV |
| **状态变更操作** | 待审批→已批准→制作中→已发货→已完成，支持跳过制作 |
| **物流信息** | 发货时记录承运商和运单号 |
| **行业管理** | 创建行业、配置专属字段（文本/选择/数字/日期/开关） |
| **查询设置** | 配置前台查询功能的启用/停用 |
| **风格设置** | 配置前台页面的主题色、背景色、导航样式 |
| **进度看板** | 按状态分组查看所有申请 |
| **客户通知** | 状态变更自动记录通知，客户查询可见 |

## 十、设计原则

1. **前台无需登录** — 客户直接提交申请，降低使用门槛
2. **多行业可配置** — 不同行业通过后台配置实现，无需修改代码
3. **全流程可追溯** — 每次状态变更记录时间、操作人、备注
4. **状态标签使用颜色+文字**，色盲友好
5. **操作按钮使用动词**（批准/驳回/发货），而非模糊的确认
